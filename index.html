Concepts

What Are DNS & DHCP?

📖
DNS — The Network Phone Book
Translates human-readable names like winserver2022.homelab.local into IP addresses computers use to communicate. Without DNS, you'd need to remember every IP address.
🏨
DHCP — The Automatic IP Assigner
Automatically assigns IP addresses, subnet masks, gateways, and DNS servers to devices when they join the network — like a hotel front desk handing out room keys.
🏢
Real-World Use
Every company network uses both. DNS lets employees reach internal apps by name. DHCP ensures laptops, phones, and printers get correct network settings automatically.
🔗
AD Integration
In enterprise environments, DNS and DHCP are integrated with Active Directory — allowing dynamic updates, secure zone transfers, and authorized DHCP servers only.

Walkthrough

Step-by-Step Implementation

01
Install DHCP Server Role
DNS was already installed with AD DS. DHCP was installed separately using PowerShell with management tools included.
PowerShell — Admin
# Install DHCP Server role with management tools
Install-WindowsFeature -Name DHCP -IncludeManagementTools
✓ Result
Success: True  |  Restart Needed: No  |  Feature: {DHCP Server, DHCP Server Tools}
02
Authorize DHCP & Create Scope
Authorized the DHCP server in Active Directory to prevent rogue DHCP servers. Created a scope with IP range 192.168.1.50–150 and set scope options.
PowerShell — Admin
# Authorize DHCP in Active Directory
Add-DhcpServerInDC -DnsName "WinServer2022" -IPAddress 192.168.1.200

# Create DHCP scope
Add-DhcpServerv4Scope -Name "HomeLabScope" `
  -StartRange 192.168.1.50 `
  -EndRange 192.168.1.150 `
  -SubnetMask 255.255.255.0 `
  -Description "Home Lab DHCP Scope" `
  -State Active

# Set gateway and DNS options
Set-DhcpServerv4OptionValue -ScopeId 192.168.1.0 `
  -Router 192.168.1.1 `
  -DnsServer 192.168.1.200 `
  -DnsDomain "WinServer2022"
✓ DHCP Scope Summary
Name: HomeLabScope  |  Range: 192.168.1.50–150
Gateway: 192.168.1.1  |  DNS: 192.168.1.200  |  State: Active
03
Create DNS Zones
Created a primary forward lookup zone for homelab.local and a reverse lookup zone for the 192.168.1.0/24 network.
PowerShell — Admin
# Forward lookup zone
Add-DnsServerPrimaryZone -Name "homelab.local" `
  -ZoneFile "homelab.local.dns" `
  -DynamicUpdate NonSecureAndSecure

# Reverse lookup zone
Add-DnsServerPrimaryZone -NetworkId "192.168.1.0/24" `
  -ZoneFile "1.168.192.in-addr.arpa.dns" `
  -DynamicUpdate NonSecureAndSecure
04
Add DNS Records & Verify
Added an A record for the server with automatic PTR creation, a CNAME alias, and verified full DNS resolution.
PowerShell — Admin
# A record with PTR
Add-DnsServerResourceRecordA -Name "winserver2022" `
  -ZoneName "homelab.local" `
  -IPv4Address "192.168.1.200" `
  -CreatePtr

# CNAME alias
Add-DnsServerResourceRecordCName -Name "dc" `
  -ZoneName "homelab.local" `
  -HostNameAlias "winserver2022.homelab.local."

# Verify resolution
Resolve-DnsName "winserver2022.homelab.local" -Server 192.168.1.200
✓ DNS Resolution Test
Name: winserver2022.homelab.local
QueryType: A  |  TTL: 3600  |  IPAddress: 192.168.1.200

Troubleshooting

Common Issues & Fixes

DHCP Client Gets 169.254.x.x
Cause
Device couldn't reach the DHCP server and Windows assigned a self-generated APIPA address instead.
✓ Fix
Verify DHCP service is running and the scope is active. Check network connectivity between client and server.
DNS Not Resolving Internal Names
Cause
Client is using external DNS (router) instead of the internal DNS server at 192.168.1.200.
✓ Fix
Ensure DHCP scope option 006 (DNS Server) is set to 192.168.1.200 so clients receive correct DNS automatically.
DHCP Scope Exhausted
Cause
All 101 IP addresses in the pool (50–150) have been assigned and no more are available.
✓ Fix
Expand the scope range or shorten the lease duration so IPs are recycled faster. Monitor with Get-DhcpServerv4Lease.
Rogue DHCP Server Conflict
Cause
Another unauthorized DHCP server is responding to client requests with incorrect network settings.
✓ Fix
AD authorization blocks unauthorized DHCP servers automatically — this is why we ran Add-DhcpServerInDC.
